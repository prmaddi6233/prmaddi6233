# Repository Guidance

## Purpose

This repository is the personal GitHub profile repository for Pradeep Maddi.
Its primary artifact is `README.md`, which appears on the GitHub profile page
for `prmaddi6233`.

## Professional Positioning

Position Pradeep as a Cloud Platform Engineer building secure, scalable, automated, and cost-efficient AWS platforms.

The profile should support opportunities in:

- Senior Cloud Platform Engineering
- Platform Engineering
- AWS Cloud Architecture
- Cloud FinOps Engineering
- Cloud automation and governance consulting

## Content Integrity

Do not invent experience, metrics, employers, customer names, project names,
savings percentages, incident details, account IDs, certification dates, badge
IDs, or expiry dates.

Use only information that is already present in the repository or explicitly
provided by Pradeep. When adding outcomes, keep them outcome-oriented but
general enough to avoid confidential or unsupported claims.

Do not include:

- Employer names
- Customer names
- AWS account IDs
- Internal role names
- Confidential project names
- Exact company spending
- Internal incident information
- Unsupported percentages or savings numbers

## README Style Standards

Keep the README professional, senior-level, concise, technical, and easy to scan.

Prefer:

- GitHub-flavored Markdown
- Clear heading hierarchy
- Capability-based technology grouping
- Limited, reliable shields.io badges
- Outcome-oriented bullets
- Placeholder project links clearly marked as `Coming soon`

Avoid:

- Animated typing banners
- Visitor counters
- Excessive emojis
- Dozens of badges
- Joke widgets
- Unmaintained widgets
- Broken image URLs
- Skill percentages
- Self-assigned proficiency ratings
- Large HTML layouts unless Markdown cannot provide the required result

## Validation Expectations

Before completing future updates:

- Inspect every modified file.
- Check Markdown heading hierarchy.
- Check for broken local links.
- Check for malformed badge and image URLs.
- Confirm GitHub and LinkedIn links are correct.
- Confirm no confidential information was introduced.
- Run available Markdown validation locally.
- Review the Git diff.

## Future Codex Update Instructions

Keep this repository separate from organization-owned or employer-related
repositories. Do not copy internal repository names, tickets, branch names,
account details, or project details into this profile.

When updating featured projects, only replace `Coming soon` once the repository
exists and the link is verified. If a project is private, describe it
generically without linking to confidential material.

Do not push directly to `main` unless Pradeep explicitly asks for that. Prefer feature branches and pull requests.
